# Energy Trade Inspection Platform — Design System

Generated by /plan-design-review on 2026-04-04

---

## Product Character

B2B compliance tool for energy traders. Users come with a specific target in mind and mild anxiety — they need to know if a counterparty is real, sanctioned, or fraudulent. The design must communicate:
- **Trust and stability** (not startup-fresh, not corporate-sterile)
- **Professional density** (information-rich, like Bloomberg terminal, not like Notion)
- **Clarity under pressure** (the most important answer — sanction status — must be unmissable)

Classifier: **HYBRID** — marketing landing page + dense data application

Two typefaces maximum.

```
Data/Scores: JetBrains Mono — authenticity scores, registration numbers, IMO numbers, dates
```

Scale:

## Component Vocabulary

### Sanction Status Badge (most important component)
```
NOT LISTED: green circle (8px) + "NOT LISTED" text — #10b981 bg at 15% opacity, green text
LISTED:     red circle (8px) + "LISTED" text — #ef4444 bg at 15% opacity, red text
UNKNOWN:    gray circle (8px) + "UNKNOWN" — neutral

Size: always 14px text, always visible at F1 hierarchy level.
NEVER mix sanction status into the authenticity score display.
```

### Risk Level Badge
```
Low:      #10b981 bg 15% opacity + green text
Medium:   #f59e0b bg 15% opacity + amber text
High:     #f97316 bg 15% opacity + orange text
Critical: #ef4444 bg 15% opacity + red text

No colored left borders. Background fill only.
```

### Authenticity Score Display
```
Outer ring: circular gauge 0-100, color follows risk level
Score number: 48px JetBrains Mono Bold
Below score: risk tier label (Verified / Mostly Verified / Partially Verified / Insufficient / Suspicious)

Sub-dimension bars (horizontal):
  ├── Entity Existence    (max 25): [████████████░░░░░░] 18/25
  ├── Asset Reality       (max 30): [██████████████████] 22/30
  ├── Trading Track Record(max 25): [⌛ Phase 2 pending] --/25  ← amber color, not 0
  ├── Document Consistency(max 10): [███████░░░] 7/10
  └── Community Reputation(max 10): [████████░░] 8/10

Note below: "Score based on Phase 1 available data (max 75)"
```

### Risk Flag Card
```
Background: var(--bg-elevated)
Left visual: category icon (no border-left stripe)
Category badge: amber/orange depending on severity
Fields: Category | Submission date | Status (pending/verified)
Text: "Submitted anonymously — identity not disclosed"
DO NOT show submitter identity even in admin view (UI layer)
```

### Content Lock (Paid Layer)
```
Trigger: F3 content (Registration details, Directors list, Vessels list, Documents)
NOT triggered on: F1 (entity name, score, sanction badge) and F2 (summary, basic info)
→ Free users see F2 fully. F3 is blurred.

Method: CSS blur(4px) + overlay gradient + upgrade CTA inline
NOT: completely hidden (breaks SEO/GEO crawlability)
CTA text: "Full details available on Starter plan ($99/mo)" with [Upgrade] button
Position: overlaid on blurred content, not below it

Screen reader: blurred F3 content has aria-hidden="true"
CTA button: aria-label="Upgrade to Starter plan to unlock full registration details"
```

### Empty States
```
Style: Line icon (16px, --text-secondary color) + 2-line text explanation
Icon selection: matches context (search icon for "no results", file icon for "no data", clock icon for "pending")
Primary action button: always present

Examples:
  No search results:
    Icon: search
    Title: "No matches found"
    Body: "Try a different name, registration number, or IMO number."
    CTA: [Search tips]

  Entity data unavailable:
    Icon: file-question
    Title: "No registry data found"
    Body: "This may be a newly incorporated entity, or registered under a different identifier."
    CTA: [Submit a verification request]

  Directors tab empty:
    Icon: users
    Title: "No director records"
    Body: "No public director records found in available registries."
    CTA: none (informational only)
```

## Layout Rules

### Entity Profile Page
- Max content width: 1200px, centered
- Sidebar layout: 280px left sidebar (score + status) + main content area
- Tab navigation for F3 content (Registration / Directors / Vessels / Flags / Sources)
- Sticky header on scroll: entity name + sanction badge always visible

### Data Tables (Directors, Vessels, Risk Flags)
- Dense: 40px row height
- Alternating row backgrounds: --bg-surface / --bg-elevated
- Sortable columns where relevant
- No decorative card wrappers around table rows

### Search / Disambiguation
- Cards in disambiguation: 100% width, stacked, not grid
- Each card: jurisdiction flag + entity name + registration number + score badge + status badge
- Max 10 candidates shown, "Load more" below

## Responsive Breakpoints

```
Desktop:  1280px+ — sidebar layout, full density
Tablet:   768-1279px — sidebar collapses to top bar, reduced columns
Mobile:   < 768px — single column, score ring collapses to horizontal bar, tabs become dropdown
```

### Desktop (1280px+)
- 280px left sidebar (Score ring + Sanction badge + Risk level) + main content
- Full data table density (40px rows, all columns visible)
- Tab navigation inline, sticky on scroll

### Tablet (768-1279px)
- Sidebar collapses: Score + Sanction badge move to a 64px top bar (horizontal layout)
  - Score: compact circular gauge (64px diameter) + score number inline
  - Sanction badge: right-aligned in top bar
- Main content area: full width
- Data tables: hide lowest-priority columns (e.g., hide "Source" column in Directors table)
- Tab navigation: full width, horizontal scroll if overflow
- Top bar sticks on scroll

### Mobile (< 768px)
- Single column layout
- Score display: horizontal progress bar (full width) + score number + risk label inline
  - Height: 8px bar, same color logic as circular gauge
- Sanction badge: full-width pill below score bar
- Tab navigation: horizontal scroll (not hamburger, not dropdown)
  - Each tab: minimum 80px width, 44px height touch target
- Risk badges: always visible, never collapsed
- Search box: full width, 48px height (touch target)
- Risk Flag cards: full width, reduced padding (16px)
- Disambiguation cards: full width, stacked, 16px gap

Mobile-specific rules:
- Search box: full width, 48px height (touch target)
- Score display: horizontal bar instead of circular gauge
- Tab navigation: horizontal scroll, not hamburger
- Risk badges: always visible (never hidden on mobile)
- "Load more" button in disambiguation: full width, 48px height, sticky-ish (not fixed)

## Accessibility

- Color contrast: minimum AA (4.5:1) for all text on dark backgrounds
- Touch targets: minimum 44x44px for all interactive elements
- Keyboard navigation: Tab order follows F1 → F2 → F3 hierarchy
- Screen reader: aria-label on all badges ("Sanction status: Not listed"), score description ("Authenticity score: 55 out of 100")
- Focus indicators: 2px blue outline (#3b82f6), never hidden
- Status colors never rely on color alone — always paired with icon or text

### Keyboard Navigation Details

```
Homepage:
  Tab 1 → Search input
  Tab 2 → Entity type selector (Company / Vessel / Terminal)
  Tab 3 → Search button
  Tab 4+ → Sample entity cards (if visible)

Entity Profile Page:
  Tab 1 → Sticky header entity name (skip link to main content)
  Tab 2 → Sanction badge (focusable, reads: "Sanction status: Not listed")
  Tab 3 → Tab navigation (Registration / Directors / Vessels / Flags / Sources)
           ↳ Arrow keys navigate between tabs
  Tab 4+ → Content within active tab
           ↳ Data table rows: focusable via Tab, not individual cells
           ↳ Each row reads: "[Entity name], [key field], [status]"

Data Tables:
  - Table rows are focusable (tabindex="0")
  - Individual cells are NOT focusable (reduces Tab noise)
  - Sort buttons in column headers: focusable, announces "Sort by [column], [ascending/descending]"
  - "Load more" button at table bottom: focusable

Content Lock:
  - Blurred content: aria-hidden="true" (screen reader skips)
  - Upgrade CTA button: visible, focusable, aria-label="Upgrade to Starter plan to view full details"
```

### Screen Reader Specifics

```
Score Gauge (SVG):
  - Wrap in <figure> with <figcaption class="sr-only">Authenticity score: 55 out of 100. Rating: Partially Verified.</figcaption>
  - SVG itself: aria-hidden="true"

Score Sub-dimension bars:
  - Each bar: role="progressbar" aria-valuenow="18" aria-valuemax="25" aria-label="Entity Existence score"
  - Phase 2 pending bars: aria-label="Trading Track Record: data pending Phase 2" aria-valuenow="0" aria-valuemax="25"

Sanction Badge:
  - role="status" aria-live="polite" aria-label="Sanction status: Not listed on any sanctions database"

Risk Flag Cards:
  - role="article" aria-label="Risk flag: [category], submitted [date], status [pending/verified]"
```

## Internationalization (i18n)

**Decision: Bilingual — CN/EN with manual toggle (from Phase 1)**

```
- Language toggle: right-top corner, "EN | 中文" pill switcher
- Default: detect from Accept-Language header (zh → 中文, others → EN)
- User can override: selection persisted in localStorage
- i18n approach: standard i18n library (next-intl / i18next)
- All UI strings in message files — no hardcoded English text in components
- Entity data (company names, descriptions): displayed as-is from data source
  (some source data is English-only — do not auto-translate data fields)
- GEO content (entity pages, Schema.org metadata): English only for AI crawler indexing
```

UI copy split:
- Navigation, labels, CTAs, error messages, empty states → i18n translated
- Raw data from sources (registration numbers, addresses) → displayed as-is
- Legal disclaimers → bilingual (both languages visible simultaneously)

## GEO-Specific Design Rules

Entity pages are designed for dual audiences: human users AND AI crawlers.

- All text content in `<p>`, `<h1-h3>` tags, never image-only
- Sanction status and risk level in visible text, not only visual badges
- `<time datetime="2026-04-01">` for all data timestamps
- Source attribution in visible text (not tooltip-only)
- Score tier ("Partially Verified") in text alongside the visual gauge
