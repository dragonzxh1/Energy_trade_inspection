## API 数据源选择

| Option | Description | Selected |
|--------|-------------|----------|
| data.gov.hk 开放数据 | 类似 ACRA 模式，免费、正规、可批量下载。创建独立的 hkcr.ts 同步模块。局限：可能无董事数据 | ✓ |
| OpenCorporates HK 专项层 | 在现有 OC 集成基础上，将 HK 管辖区查询从通用层提升为专项优先级。少量新代码，但依赖 OC API（限速） | |
| 两者结合 | data.gov.hk 作为主力，OpenCorporates hk 作为备用层。实现最全面但工作量最大 | |

**User's choice:** data.gov.hk 开放数据（推荐）
**Notes:** 与 ACRA 使用 data.gov.sg 完全类比；OpenCorporates hk 作为隐式备用（已集成，无需额外代码）

## 捕获数据字段范围

| Option | Description | Selected |
|--------|-------------|----------|
| 基础字段 | 公司名、注册号、状态、注册日期、公司类型。与 ACRA 同级别，不需要额外 API 授权 | ✓ |
| 扩展字段（含董事/公司秘书） | 在基础字段基础上加董事和公司秘书。可能需要求评个层级 API，或需要 OpenCorporates 升级计划 | |

**User's choice:** 基础字段（推荐）
**Notes:** 董事/公司秘书字段延后至有付费 API 计划时

---

## SFC 联动方式

| Option | Description | Selected |
|--------|-------------|----------|
| 显示层对应 | 搜索到香港实体时，同时查询 warning_list_entries 表（SFC 来源）是否有同名条目，若匹配则附带 WarningBadge。无需新迁移或 JOIN 外键 | ✓ |
| 数据库结构性关联 | 在 entities 或缓存表中新增外键直接关联 SFC warning 条目 ID。更强但需要 migration 和同步逻辑 | |

**User's choice:** 显示层对应（推荐）
**Notes:** 与其他 registry 来源（ACRA/CH）结果的 SFC 检查方式统一

---

## 注册号格式检测

| Option | Description | Selected |
|--------|-------------|----------|
| 添加号码检测函数 | 在 hkcr.ts 中添加 mightBeHKNumber()（类似 companies-house.ts 的 mightBeUKNumber）。香港公司号为纯 7 位数字。当查询匹配时，直接优先以号码查询 | ✓ |
| 不添加格式检测 | 所有查询都走名称搜索路径。更简单但搜索精度较低 | |

**User's choice:** 添加号码检测函数（推荐）
**Notes:** `const HK_NUMBER_REGEX = /^\d{7}$/`，7位纯数字
