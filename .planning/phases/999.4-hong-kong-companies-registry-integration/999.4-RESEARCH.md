# Phase 999.4: 香港公司注册处接入 — Research

**Researched:** 2026-04-19
**Domain:** Hong Kong Companies Registry data integration via open data APIs
**Confidence:** MEDIUM (API availability constraint discovered)

## Summary

本 Phase 的核心发现：**data.gov.hk CKAN API 不提供可搜索的完整香港公司注册数据库**。与新加坡 ACRA（data.gov.sg 提供完整的 UEN 实体搜索 API）不同，香港 Companies Registry 在 data.gov.hk 上仅发布**每周新增公司名单**（weekly list of newly incorporated companies），不包含历史存量数据的搜索接口。

ICRIS 系统（icris.cr.gov.hk）是官方搜索界面，但带有 anti-bot 保护（disable-devtool.min.js），非免费 API。

**Primary recommendation:** 调整实现策略 — OpenCorporates 已覆盖香港 (`hk` jurisdiction code)，应作为 HK 实体的主要数据源。若需增强 HK 覆盖，可创建 `searchHKOC()` 函数专门筛选香港司法管辖区的 OC 结果，而非尝试接入不存在于 data.gov.hk 的可搜索 API。

## Critical Finding: data.gov.hk API Availability

### Investigation Results

**Query 1:** `https://data.gov.hk/en-data/api/3/action/package_search?q=company&rows=10`

[VERIFIED: curl request 2026-04-19] 返回 38 个数据集，其中唯一来自 `hk-cr` (Companies Registry) 组织的数据集为：

```
"name": "hk-cr-crdata-list-newly-registered-companies-2526"
"title": "List of Newly Incorporated / Registered / Re-domiciled Companies and Companies which have changed Names (2024.12.30 - Current)"
"num_resources": 268
"notes": "List of Newly Incorporated / Registered / Re-domiciled Companies and Companies which have changed Names"
```

**关键发现：**
- 该数据集仅包含**每周新增公司名单**，不是可搜索的完整注册数据库
- 每周更新，资源数量 268（每个周一份文件）
- 无法通过 `q` 参数搜索历史公司

**Query 2:** `https://data.gov.hk/en-data/api/3/action/organization_show?id=hk-cr&include_datasets=true`

[VERIFIED: curl request 2026-04-19] HK CR 组织仅发布上述一个数据集，无其他可搜索的公司注册数据集。

**Query 3:** ICRIS 系统 `https://www.icris.cr.gov.hk/csci/`

[VERIFIED: curl request 2026-04-19] 该页面包含：
- CSRF token 保护
- `disable-devtool.min.js` anti-bot 脚本
- 需要 JavaScript 渲染的 SPA 界面

**结论：** 香港 Companies Registry 无公开的免费搜索 API。官方 ICRIS 系统是 Web 界面，非 API。

## Common Pitfalls

### Pitfall 1: 误用 data.gov.hk datastore_search

**What goes wrong:** 尝试 `datastore_search?q={query}` 搜索香港公司，返回空结果
**Why it happens:** data.gov.hk 的 HK CR 数据集仅包含每周新增公司列表（CSV/XLSX 文件），非可搜索数据库
**How to avoid:** 研究阶段验证 API 可用性，确认数据集结构后再规划实现
**Warning signs:** CKAN package_search 返回 `num_resources: 268`（每个周一份），而非单一 datastore resource

### Pitfall 2: HK 注册号与 UK 注册号混淆

**What goes wrong:** 7 位数字可能是香港公司号，也可能是 UK 公司号（SC/NI 前缀时）
**Why it happens:** UK 的 SC/NI 前缀格式如 `SC123456`，纯数字时与 HK 格式重叠
**How to avoid:** 精确 regex：HK 为 `^\d{7}$`，UK 为 `^(SC|NI|...)\d{6,8}$`
**Warning signs:** Companies House 返回匹配但 `jurisdiction_code = 'gb'`

### Pitfall 3: SFC 警告名称标准化不一致

**What goes wrong:** 公司名与 SFC 警告名匹配失败（`BP Trading` vs `BP TRADING LTD`）
**Why it happens:** 未使用相同的标准化函数（normalizeEntityName）
**How to avoid:** [VERIFIED: warning-lists.ts line 33] 使用 `normalizeEntityName(entityName, false)` 保持词序
**Warning signs:** 模糊匹配 similarity < 0.72 threshold

### SFC Warning Check Integration

```typescript
// Source: warning-lists.ts line 25-67
export async function getWarningHits(
  entityName: string,
  _entityType?: 'company' | 'vessel' | 'terminal'
): Promise<WarningHit[]> {
  const normalizedQuery = normalizeEntityName(entityName, false)
  
  const { rows } = await db.query<WarningHit & { similarity: number }>(
    `SELECT source, source_name, jurisdiction, entity_name, list_url, warning_type,
       word_similarity($1, normalized_name) AS similarity
     FROM regulatory_warnings
     WHERE word_similarity($1, normalized_name) >= 0.72
     ORDER BY similarity DESC`,
    [normalizedQuery]
  )
  
  // Deduplicate by source
  const bySource = new Map<string, WarningHit>()
  for (const row of rows) {
    if (!bySource.has(row.source)) {
      bySource.set(row.source, {
        source: row.source,
        source_name: row.source_name,
        jurisdiction: row.jurisdiction,
        entity_name: row.entity_name,
        list_url: row.list_url,
        warning_type: row.warning_type,
      })
    }
  }
  return Array.from(bySource.values())
}

// SFC entries are synced via regulatory-warnings.ts line 172-197 (scrapeSfc function)
```
