# Phase 999.4: 香港公司注册处接入 — Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

接入香港公司注册处 (Companies Registry) 数据，将香港实体的注册信息整合到 ETI 搜索和情报层：

1. **hkcr.ts 同步模块**：通过 data.gov.hk 开放数据 API（CKAN 模式）按名称搜索香港注册公司
2. **repository.ts Tier 2 接入**：`searchHKCR()` 与 ACRA/Companies House/Zefix/OC 并行，返回带 `registrySource: 'hkcr'` 标记的实体
3. **SFC 警告联动**：搜索到香港实体时，在显示层检查 `warning_list_entries` 表（SFC 来源），若匹配则附带 WarningBadge
4. **注册号检测**：`mightBeHKNumber()` 函数识别 7 位纯数字查询，优先走号码查询路径

**本 Phase 不包含**：董事/公司秘书字段（需高级 API）、数据库结构性外键关联（SFC 联动仅在显示层）、HK 实体的全量批量同步到本地表（走 Tier 2 实时查询，不缓存）。

</domain>

<decisions>
## Implementation Decisions

### D-01: API 数据源 — data.gov.hk 开放数据（CKAN 接口）

**方案：** 创建独立 `src/lib/server/sync/hkcr.ts`，使用 `data.gov.hk` CKAN API 搜索香港注册公司。模式完全类比 `acra.ts`（同为政府开放数据 CKAN 接口）。

- **主接口格式：** `https://data.gov.hk/en-data/api/3/action/datastore_search?resource_id={RESOURCE_ID}&q={query}`
- **无需 API Key**：data.gov.hk 开放数据免费，无需注册（类比 ACRA）
- **OpenCorporates hk 作为备用**：若 data.gov.hk 无数据或服务不可用，现有 `searchOC()` 已覆盖 `hk` 司法管辖区，自动兜底
- **不使用** HK CR 官网（需 CAPTCHA）、不使用付费 HKEX API

### D-02: 捕获字段范围 — 基础字段（与 ACRA 同级）

```typescript
export interface HKCRCompany {
  company_number: string          // 7位注册号
  company_name: string            // 公司英文名
  company_name_chinese?: string   // 公司中文名（若 API 提供）
  company_type: string            // e.g. 'Private Company Limited by Shares'
  company_status: string          // 'Live' | 'Deregistered' | 'Dissolved'
  date_of_incorporation: string   // ISO 日期
  nature_of_business?: string     // 业务性质
}
```

**不捕获**：董事、公司秘书、注册地址（data.gov.hk 开放数据集通常不含这些字段；如需扩展，后续阶段可接入 OpenCorporates 付费 officers 端点）。

### D-03: SFC 监管警告联动 — 显示层对应，无结构变更

联动策略：**不新增迁移，不新增外键**。

- 搜索到香港实体后，在 `repository.ts` 的结果构建阶段，检查 `warning_list_entries` 表中 `source = 'sfc'` 的条目是否与该公司名匹配
- 若匹配，在返回的搜索结果对象中附带 `hasWarning: true` + `warningSource: 'sfc'`（复用现有 WarningBadge 展示机制）
- 与现有 ACRA/Companies House 结果的 SFC 检查方式一致（统一模式，不需 HK 专属逻辑）

### D-04: 注册号格式检测 — mightBeHKNumber()

```typescript
// 香港公司注册号：7位纯数字（无前缀字母）
const HK_NUMBER_REGEX = /^\d{7}$/

export function mightBeHKNumber(s: string): boolean {
  return HK_NUMBER_REGEX.test(s.trim())
}
```

- 类比 `companies-house.ts` 的 `mightBeUKNumber()`
- 当查询字符串匹配时，`searchHKCR()` 优先调用按号码精确查找接口（若 CKAN API 支持），次选名称搜索

### Claude's Discretion

- data.gov.hk 的具体 resource_id（需在实现时确认最新数据集 ID，CR 公司数据集）
- CKAN API 的确切参数格式（`q` 参数 vs `filters` 参数）
- 批次大小和 `fetch()` 超时设置（参考 acra.ts 的 5000ms）
- 错误处理：若 data.gov.hk 不可用时如何优雅降级（返回空数组，让 OC fallback 生效）
- `searchHKCR()` 在 `repository.ts` Tier 2 中的并行位置（加入 `Promise.all` 数组）

</decisions>

<specifics>
## Specific Ideas

- **data.gov.hk 数据集 ID**：HK CR 在 data.gov.hk 上的公司注册数据集需在实现前确认（URL 格式：`https://data.gov.hk/en-data/dataset/hkgov-commerce-cr-registered-companies`）
- **中英文名并存**：香港公司同时有英文名和中文名，CKAN API 可能两者都返回，`company_name_chinese` 字段可选存储
- **注册号 7位**：Hong Kong CR 的公司注册号为 7 位纯数字（如 `1234567`），本地分支机构可能有不同格式，初期只处理 7 位
- **Tier 1 vs Tier 2**：与 ACRA 一致，hkcr 走 Tier 2（实时外部 API 调用），若 Tier 1 已有本地命中则跳过

</specifics>

<deferred>
## Deferred Ideas

- **董事/公司秘书字段**：需要 OpenCorporates 付费 officers 端点或 HK CR 付费 API，延后至有付费计划时
- **全量本地缓存**：将 HK 公司批量下载到本地表（类似 lei_cache），当前 Tier 2 实时查询已满足需求，批量缓存是后续优化
- **HKEX 上市公司专项集成**：香港上市公司通过 HKEX API 可获取更丰富数据，是独立的未来 Phase
- **SFC 数据库级外键关联**：用外键直接连接 HK 注册实体与 SFC 警告条目，当前显示层联动已满足需求

</deferred>
