# Phase 999.4: 香港公司注册处接入 — Context

**Gathered:** 2026-04-19
**Updated:** 2026-04-19 (策略调整：CSV批量下载 + 本地缓存)
**Status:** Ready for planning

<domain>
## Phase Boundary

接入香港公司注册处 (Companies Registry) 数据，将香港实体的注册信息整合到 ETI 搜索和情报层：

1. **hkcr.ts 同步模块**：从 data.gov.hk 批量下载 CSV 文件（每周新增公司名单），解析并导入本地缓存表 `hkcr_cache`
2. **hkcr_cache 表**：创建 migration 存储香港公司数据（公司名、注册号、状态、注册日期等）
3. **repository.ts Tier 2 接入**：`searchHKCRCache()` 从本地缓存查询，返回带 `registrySource: 'hkcr'` 标记的实体
4. **SFC 警告联动**：搜索到香港实体时，在显示层检查 `warning_list_entries` 表（SFC 来源），若匹配则附带 WarningBadge
5. **注册号检测**：`mightBeHKNumber()` 函数识别 7 位纯数字查询，优先走号码精确查询路径

**关键调整（2026-04-19）**：
- data.gov.hk CKAN API **不提供可搜索的公司数据库**（仅提供每周新增 CSV 文件）
- OpenCorporates **无免费层**，不作为数据源
- 实现策略从"实时API查询"调整为"CSV批量下载 + 本地缓存查询"

**本 Phase 不包含**：董事/公司秘书字段（需高级 API）、数据库结构性外键关联（SFC 联动仅在显示层）。

</domain>

<decisions>
## Implementation Decisions

### D-01: 数据源策略 — CSV批量下载 + 本地缓存

**方案：** 从 data.gov.hk CKAN API 下载每周新增公司 CSV 文件，解析后导入本地缓存表 `hkcr_cache`。

- **数据源：** `https://data.gov.hk/en-data/api/3/action/package_show?id=hk-cr-crdata-list-newly-registered-companies-2526`
- **CSV 文件数量：** 268 个资源（每周一份，从 2024.12.30 至今）
- **无需 API Key**：data.gov.hk 开放数据免费
- **本地缓存表：** `hkcr_cache`（类比 `lei_cache` 模式）
- **查询路径：** Tier 2 从 `hkcr_cache` 表查询，而非实时 API 调用
- **不使用** OpenCorporates（无免费层）、HK CR 官网（需 CAPTCHA）、付费 HKEX API

**实现类比：** `gleif-golden-copy.ts` 的批量下载 + 解析 + 导入模式，而非 `acra.ts` 的实时 CKAN 搜索模式。

### D-02: 捕获字段范围 — 基础字段（与 ACRA 同级）

```typescript
export interface HKCRCompany {
  company_number: string          // 7位注册号
  company_name: string            // 公司英文名
  company_name_chinese?: string   // 公司中文名（若 API 提供）
  company_type: string            // e.g. 'Private Company Limited by Shares'
  company_status: string          // 'Live' | 'Deregistered' | 'Dissolved'
  date_of_incorporation: string   // ISO 日期
  nature_of_business?: string     // 业务性质
}
```

**不捕获**：董事、公司秘书、注册地址（data.gov.hk 开放数据集通常不含这些字段；如需扩展，后续阶段可接入 OpenCorporates 付费 officers 端点）。

### D-03: SFC 监管警告联动 — 显示层对应（无需新增代码）

联动策略：复用现有 `getWarningHits()` 函数，无需新增代码。

- **实现位置：** `src/app/company/[slug]/page.tsx` 等实体详情页（显示层），而非 `repository.ts` 搜索层
- **机制：** 用户查看香港实体详情时，page.tsx 自动调用 `getWarningHits(company.name, 'company')`，返回 SFC 警告匹配
- **数据来源：** SFC 数据已通过 `regulatory-warnings.ts` 同步到 `warning_list_entries` 表（source='sfc'）
- **一致性：** 与 ACRA/Companies House 结果的 SFC 检查方式完全一致（同一调用位置）
- **本 Phase 责任：** 确保 HKCR 数据可被搜索，SFC 警告联动由现有基础设施自动处理

### D-04: 注册号格式检测 — mightBeHKNumber()

```typescript
// 香港公司注册号：7位纯数字（无前缀字母）
const HK_NUMBER_REGEX = /^\d{7}$/

export function mightBeHKNumber(s: string): boolean {
  return HK_NUMBER_REGEX.test(s.trim())
}
```

- 类比 `companies-house.ts` 的 `mightBeUKNumber()`
- 当查询字符串匹配时，优先从 `hkcr_cache` 表按 `company_number` 精确查询，次选名称模糊查询

### D-05: Migration — hkcr_cache 表

**新增 migration：** `db/migrations/038_hkcr_cache.sql`

```sql
CREATE TABLE hkcr_cache (
  id SERIAL PRIMARY KEY,
  company_number TEXT NOT NULL UNIQUE,
  company_name TEXT NOT NULL,
  company_name_chinese TEXT,
  company_type TEXT,
  company_status TEXT,
  date_of_incorporation DATE,
  nature_of_business TEXT,
  fetched_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_hkcr_cache_name ON hkcr_cache USING pg_trgm (company_name);
CREATE INDEX idx_hkcr_cache_number ON hkcr_cache (company_number);
```

### Claude's Discretion

- CSV 文件的具体字段格式（需在实现时从下载的 CSV 确认）
- CSV 解析库选择（csv-parse 或 papaparse）
- 批量下载的并发控制（防止服务器拒绝）
- `searchHKCRCache()` 在 `repository.ts` Tier 2 中的并行位置
- 同步触发方式：管理员手动触发 `/api/admin/sync?source=hkcr` 或 cron job

<specifics>
## Specific Ideas

- **CSV 文件格式**：需在实现时确认字段名（如 Company_Number、Company_Name、Date_of_Incorporation 等）
- **中英文名并存**：香港公司同时有英文名和中文名，CSV 可能两者都返回，`company_name_chinese` 字段可选存储
- **注册号 7位**：Hong Kong CR 的公司注册号为 7 位纯数字（如 `1234567`），初期只处理 7 位
- **数据覆盖范围**：268 个 CSV 文件覆盖 2024.12.30 至今的新增公司，历史存量公司需后续接入付费 API
- **增量同步**：每周新增 CSV 可通过 cron job 自动拉取，追加到 `hkcr_cache` 表

</specifics>

<deferred>
## Deferred Ideas

- **董事/公司秘书字段**：需要 OpenCorporates 付费 officers 端点或 HK CR 付费 API，延后至有付费计划时
- **历史存量公司数据**：2024.12.30 之前注册的公司不在 CSV 文件中，需付费 API 或手动采购数据
- **HKEX 上市公司专项集成**：香港上市公司通过 HKEX API 可获取更丰富数据，是独立的未来 Phase
- **SFC 数据库级外键关联**：用外键直接连接 HK 注册实体与 SFC 警告条目，当前显示层联动已满足需求

</deferred>
