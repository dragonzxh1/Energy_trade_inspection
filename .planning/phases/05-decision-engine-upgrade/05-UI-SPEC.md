## Spacing Scale

All spacing uses CSS custom properties defined in `src/styles/globals.css`. Multiples of 4 throughout.

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | --space-1 | 4px | Icon gaps, tight inline spacing |
| sm | --space-2 | 8px | Badge padding, chip gaps |
| md | --space-3/4 | 12px/16px | Element padding, card internal spacing |
| lg | --space-5/6 | 20px/24px | Section padding, card gap |
| xl | --space-8 | 32px | Layout section separation |
| 2xl | --space-12 | 48px | Major section breaks |
| 3xl | --space-16 | 64px | Page-level padding |

Exceptions:
- Verdict banner: padding 20px (`var(--space-5)`) — matches existing `ResultBanner` pattern
- Flag card: padding 16px (`var(--space-4)`) — matches existing `FlagCard` pattern
- Tooltip: absolute-positioned, internal padding `8px 12px` (`var(--space-2)` vertical, `var(--space-3)` horizontal)
- FlagCard expanded explanation row: padding-top 8px (`var(--space-2)`) top border separator

Source: `src/styles/globals.css` lines 44–55. Confirmed in `TradeClient.tsx` inline style usage.

## Typography

Exactly 4 sizes, exactly 2 weights (400 / 590).

| Role | Size | Weight | Line Height | CSS var / note |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 | 22px (1.57) | Base body — flag reason text, card descriptions, verdict text |
| Label / badge | 12px | 590 | 16px (1.33) | Badge text, section headers, metadata rows, severity chips, evidence chips, tooltip list, micro/chip elements, inline labels |
| Heading (section) | 20px | 590 | 28px | Section headings (h2 and h3 unified) |
| Display / h1 | 32px | 590 | 40px | Page-level headings only |

Typography consolidation notes:
- 10px, 11px, 13px collapsed to **12px** — all chip, micro, badge, and verdict-pill text uses 12px
- 15px verdict text collapsed to **14px** — body weight (400), `var(--text-primary)`
- 24px heading collapsed to **20px** — single section heading size used for both h2 and h3 contexts
- Weight 600 mapped to **590** throughout — no component-specific notes reference 600
- Weight 700 mapped to **590** throughout — no component-specific notes reference 700
- Weight 510 removed — all former 510 usages (inline labels, FlagCard explanation title) map to **590**
- Monospace data elements (IMO numbers, registration numbers, timestamps): 12px or 14px, `var(--font-mono)`

Source: `src/styles/globals.css` lines 84–112, `TradeClient.tsx` lines 65–78 (RiskBadge), lines 529 (ResultBanner headline).

## Component Inventory

New and modified UI components for this phase:

### 1. VerdictLabel (new, inside ResultBanner)

**What:** Add a `Safe / Review / Block` verdict pill to the existing `ResultBanner` component. Placed before the `RiskBadge` in the banner header row.

**Visual spec:**
```
[BLOCK] [CRITICAL] Critical Risk — Immediate concern detected
```

Verdict pill: same structure as existing `RiskBadge large` (`fontSize: 12px`, `fontWeight: 590`, `letterSpacing: 0.06em`, `borderRadius: 4px`, `padding: 4px 12px`). Display text: `"BLOCK"` / `"REVIEW"` / `"SAFE"`. Colors from VERDICT_RISK_MAP → RISK_COLOR/RISK_BG/RISK_BORDER.

**Focal point:** The VerdictLabel pill is the primary visual anchor of the results screen — placed first in the banner row, it draws the eye before the risk level and headline text.

**Interaction:** No hover state. Static display label.

**Placement:** In `ResultBanner`, before `<RiskBadge level={overallRisk} large />`. `display: flex`, `gap: var(--space-3)`, `flexWrap: wrap`.

Source: `TradeClient.tsx` lines 505–545, CONTEXT.md specifics section.

### 2. FlagCard — expanded explanation section (updated)

**What:** Add a collapsible or always-visible explanation section below the existing evidence chips inside `FlagCard`. Shows `FLAG_EXPLANATIONS[flag.code]` content plus data source attribution.

**Visual spec (always-visible approach — no collapse toggle):**
```
[CRITICAL] [SANCTION EXPOSURE] [Seller]
Counterparty appears on OFAC SDN list ...
  [evidence chip] [evidence chip]
──────────────────────────────────
What this means: Sanctions Exposure
The counterparty or vessel appears on an official sanctions list ...
Source: OFAC SDN · Last synced: 2026-04-10
```

**Styling:**
- Separator: `borderTop: 1px solid var(--border-subtle)`, `marginTop: var(--space-2)`, `paddingTop: var(--space-2)`
- Explanation title: `fontSize: 12px`, `fontWeight: 590`, `color: var(--text-primary)`, `marginBottom: 4px` (var(--space-1))
- Explanation description: `fontSize: 12px`, `color: var(--text-secondary)`, `lineHeight: 1.5`, `margin: 0 0 4px`
- Data source line: `fontSize: 12px`, `color: var(--text-muted)`, monospace data portions use `var(--font-mono)`

**Source attribution format:** `"Source: {flag.dataSource} · Last synced: {fmt(flag.dataSourceSyncedAt) ?? 'Unknown'}"`

Source: `TradeClient.tsx` FlagCard pattern lines 258–310, CONTEXT.md D-05, RESEARCH.md Pattern 4.

### 3. SanctionBadge — tooltip upgrade (updated)

**What:** Add optional `sources?: string[]` prop. When `status === 'listed'` and `sources.length > 0`, render a JS-driven tooltip listing the specific sanctions lists.

**Visual spec:**

Badge (unchanged): `"Sanctioned"` in red (`--status-listed = #ef4444`) on `rgba(239,68,68,0.12)`, with `badge-glow-listed` pulse animation.

Tooltip (new):
```
┌─────────────────────────────┐
│ Sanctioned:                 │
│ • OFAC SDN                  │
│ • EU FSF                    │
└─────────────────────────────┘
```

Tooltip styling:
- Container: `position: absolute`, `top: calc(100% + 4px)`, `left: 0`, `zIndex: 50`
- Background: `var(--bg-elevated) = #191a1b`
- Border: `1px solid var(--border-default) = rgba(255,255,255,0.08)`
- Border radius: `6px`
- Padding: `8px 12px`
- Min-width: `160px`
- Label line: `"Sanctioned:"` — `fontSize: 12px`, `fontWeight: 590`, `color: var(--text-muted)`, `marginBottom: 4px`
- Source items: `fontSize: 12px`, `color: var(--text-primary)`, one per line or comma-separated inline

**Interaction states:**
- Desktop: `onMouseEnter` → show tooltip, `onMouseLeave` → hide tooltip
- Mobile: `onClick` → toggle tooltip
- Implementation: `useState(false)` in `SanctionBadge`, no third-party library
- Wrapper: `position: relative`, `display: inline-block`

**When to NOT show tooltip:**
- `status !== 'listed'` — no tooltip in any case
- `sources` is undefined or empty — fall back to existing `title=` attribute behavior (or omit entirely)

Source: `SanctionBadge.tsx`, `WarningBadge.tsx` (reference pattern), CONTEXT.md D-02, RESEARCH.md Pattern 5.

### 4. PDF Audit Trail — VerdictBanner (new PDF section)

**What:** New `VerdictBanner` component in `src/lib/pdf/trade-report.tsx`. Placed before existing `<RiskBanner>`.

**PDF palette (already in trade-report.tsx `C` object):**
```
block  → C.listed = #ef4444
review → C.warn   = #f97316
safe   → C.clear  = #22c55e
```

**Visual spec:**
```
┌─ Compliance Verdict ───────────────────────────────┐
│ BLOCK                                              │
│ This verdict is a compliance tool recommendation.  │
│ Authorized compliance officers are responsible ...  │
└────────────────────────────────────────────────────┘
```

Section label: `"COMPLIANCE VERDICT"` — `fontSize: 8`, `fontFamily: Helvetica-Bold`, `textTransform: uppercase`, `letterSpacing: 1`
Verdict value: `"BLOCK"` / `"REVIEW REQUIRED"` / `"SAFE TO PROCEED"` — `fontSize: 24`, `fontFamily: Helvetica-Bold`
Disclaimer text: `"This verdict is a compliance tool recommendation. Authorized compliance officers are responsible for final trade decisions."` — `fontSize: 9`, `lineHeight: 1.5`, `color: C.textSec`

Background: `{verdictColor}18` (12% alpha hex), Border: `1 solid {verdictColor}50` (31% alpha hex)

Fallback: if `result.verdict` is `undefined` (old stored JSONB rows), render nothing — do not render the section. Guard: `{result.verdict && <VerdictBanner verdict={result.verdict} />}`

Source: CONTEXT.md D-04, RESEARCH.md Pattern 7, `trade-report.tsx` lines 64–72 (riskBanner styles).

## Interaction Contract

### SanctionBadge tooltip

| Trigger | Action |
|---------|--------|
| Mouse enter (desktop) | Show tooltip |
| Mouse leave (desktop) | Hide tooltip |
| Click / tap (mobile) | Toggle tooltip open/closed |
| Click outside (optional) | Future improvement — not required this phase |
| Escape key | Not required this phase |

State: `useState<boolean>(false)` inside `SanctionBadge`. Tooltip renders conditionally — no animation required (instant show/hide is acceptable). `prefers-reduced-motion` is already handled by globals.css catch-all rule.

### FlagCard explanation

No interaction — always-visible expanded section below evidence chips. No collapse toggle in this phase. Static render.

### Verdict pill

No interaction. Static label. No click target, no tooltip.

### PDF export link

Existing `<a href download>` pattern. No change to behavior. Text change only.
