## Interaction Specification

Source: `10-CONTEXT.md` D-02 (GRAPH-02 clickable navigation), D-11 (truncation), D-12.

### Click Behavior

| Element | Action | Condition |
|---------|--------|-----------|
| Node with `etlKey` (company slug or vessel IMO) | Navigate to `/company/{slug}` or `/vessel/{imo}` using `router.push()` | Node has `etlKey` field in serialized data |
| Node without `etlKey` (director person, bare ICIJ node) | No navigation — node is non-clickable | `etlKey` is null/undefined |
| Graph canvas background | Deselect any selected node | Standard React Flow behavior |

Clickable nodes must show `cursor: pointer` on hover. Non-clickable nodes show `cursor: default`.

### Hover States

| Element | Visual Change |
|---------|--------------|
| Clickable node (hover) | Border brightens to 100% opacity; label switches from `--text-secondary` to `--text-primary`; `filter: brightness(1.15)` on fill |
| Non-clickable node (hover) | Tooltip (native `title`) shows full entity name; no border change |
| Edge (hover) | Stroke opacity increases from base to `rgba(255,255,255,0.25)` |

Hover transitions: `transition: filter 0.1s ease, border-color 0.1s ease` — matches existing `.data-row` transition timing.

### Zoom and Pan

React Flow built-in behaviors — preserve defaults:

| Behavior | Setting |
|----------|---------|
| Scroll to zoom | Enabled (default) |
| Drag to pan | Enabled (default) |
| Min zoom | 0.3 |
| Max zoom | 2.0 |
| Fit view on mount | `fitView: true` with `fitViewOptions: { padding: 0.15 }` |
| Controls | Show React Flow `<Controls>` component, positioned bottom-left |
| MiniMap | Hidden — canvas is already compact at 640px; minimap adds complexity without value |

React Flow controls background: `var(--bg-elevated)` `#191a1b`. Control button borders: `var(--border-subtle)`. Control icon color: `var(--text-muted)`.

### Accessibility

- Graph container: `role="img"` with `aria-label="Network graph showing {company name}'s ownership and director connections"`.
- React Flow keyboard navigation is not supported by default; add `tabIndex={-1}` to canvas and include a screen-reader-accessible fallback text summary listing top-level connections (directors count, vessel count, ICIJ nodes count).
- All interactive nodes: `role="button"` with `aria-label="{node name} — click to view entity"`.
- Focus ring on node focus: `outline: 2px solid var(--accent-primary)`, `outline-offset: 2px` — matches `:focus-visible` global rule.

## State Specifications

### Loading State

Triggered while `getNetworkGraph()` executes server-side. Since data is fetched in the Server Component and passed as props, there is no client-side loading for the initial render.

If the Network tab is lazy-loaded in future (Phase 11 scope), use a shimmer placeholder at canvas height 640px:

```
background: linear-gradient(90deg, var(--bg-elevated) 25%, var(--bg-subtle) 50%, var(--bg-elevated) 75%)
background-size: 400px 100%
animation: shimmer 1.4s ease infinite
```

(shimmer keyframe already defined in globals.css)

### Empty State

Triggered when: company has no directors, no vessels, no ICIJ matches (`nodes.length === 1` — only root company node).

| Element | Value |
|---------|-------|
| Heading | "No network connections found" |
| Body | "This entity has no director records, vessel associations, or ICIJ offshore matches on file." |
| Typography | Heading: 14px, `--text-primary`, weight 500. Body: 13px, `--text-muted`, line-height 20px. |
| Layout | Centered vertically and horizontally in the 640px canvas area |
| Padding | `var(--space-8)` top and bottom |

Do NOT hide the "Network" tab when empty — D-10 confirmed: always visible.

The empty state replaces the React Flow canvas. Show the outer card wrapper but render the text in place of `<ReactFlow>`.

### Truncation Banner

Triggered when `truncated === true` (ICIJ recursive query hit 100-node limit).

| Property | Value |
|----------|-------|
| Position | Fixed strip at top of canvas, overlaying graph |
| Background | `rgba(245,158,11,0.1)` (amber tint — consistent with warning treatment in codebase) |
| Border | `1px solid rgba(245,158,11,0.2)` bottom only |
| Text | "Showing 100 of {totalNodeCount} network nodes — graph truncated for performance." |
| Font | 12px, `#f59e0b` (`--accent-amber`), weight 400 |
| Padding | `var(--space-2) var(--space-4)` (8px top/bottom, 16px sides) |
| Layout | Full-width, `position: absolute; top: 0; left: 0; right: 0; z-index: 10` within canvas wrapper |

### Error State

Triggered when `getNetworkGraph()` throws or returns malformed data.

| Element | Value |
|---------|-------|
| Copy | "Unable to load network graph. Try refreshing the page." |
| Typography | 14px, `--text-muted`, centered |
| Layout | Centered in 640px canvas, same as empty state |
| Action | No retry button in Phase 10 — refresh instruction is sufficient |

### Content Lock State

The entire Network tab panel is wrapped in `<ContentLock unlocked={f3Unlocked} reason={lockReason}>`.

ContentLock behavior (existing component — do not modify):
- Blurs preview at 5px, 45% opacity, max-height 200px
- Gradient overlay from transparent to `var(--bg-surface)`
- Upgrade CTA: "Unlock Full Report" (free user) / "Sign in free" (guest)

The blurred preview that shows through the lock should be the graph canvas card itself. The NetworkGraph component renders normally; ContentLock blurs the output. This means the graph data is still computed server-side for F3-locked users — pass `nodes=[]` / `edges=[]` to NetworkGraph when `!f3Unlocked` to avoid unnecessary DB work:

```tsx
const networkGraph = f3Unlocked
  ? await getNetworkGraph(company.id)
  : { nodes: [], edges: [], truncated: false, totalNodeCount: 0 }
```

## React Flow Integration Notes

Package: `@xyflow/react` (current maintained namespace — confirmed by D-14 specifics note).
Install: `npm install @xyflow/react`.

Required React Flow imports for this component:
- `ReactFlow, Controls, Background` from `@xyflow/react`
- `@xyflow/react/dist/style.css` — import in `NetworkGraph.tsx` (scoped to component)

React Flow CSS override requirements to match project design system:

```css
/* Override React Flow defaults to match ETI dark theme */
.react-flow__background { background-color: var(--bg-surface); }
.react-flow__controls { background: var(--bg-elevated); border: 1px solid var(--border-subtle); border-radius: 6px; }
.react-flow__controls-button { background: transparent; border-bottom: 1px solid var(--border-subtle); color: var(--text-muted); }
.react-flow__controls-button:hover { background: var(--bg-subtle); color: var(--text-primary); }
.react-flow__edge-path { stroke-linecap: round; }
```

Apply these overrides via a `<style>` tag inside `NetworkGraph.tsx` or a scoped CSS module (not globals.css — avoids polluting global scope with React Flow overrides).

Background pattern: Use React Flow `<Background variant="dots">` with:
- `color: rgba(255,255,255,0.04)` (very subtle dot grid — matches dark theme)
- `gap: 20`
- `size: 1`
