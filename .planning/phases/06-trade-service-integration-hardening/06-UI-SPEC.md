## Color

All values from `globals.css` `:root` block (lines 1–56) and `TradeClient.tsx`
RISK_* maps (lines 14–31).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #08090a (`--bg-primary`) | Page background |
| Secondary (30%) | #0f1011 (`--bg-surface`) / #191a1b (`--bg-elevated`) | Cards, form container, flag cards, PartyCard, amber warning box |
| Accent — brand (10%) | #5e6ad2 (`--accent-primary`) | Submit button background ("Run Trade Check") only |
| Accent — amber | #eab308 / rgba(234,179,8,*) | Amber warning box border, background, title text — reserved for sanction degradation warning exclusively |
| Destructive | #ef4444 (`--status-listed`, `--risk-critical`) | Required field error indicator, BLOCK verdict badge, CRITICAL severity flags |

Accent reserved for:
1. `--accent-primary` (#5e6ad2): "Run Trade Check" submit button background only.
2. Amber (#eab308 / `--accent-amber`): sanction degradation warning box ONLY — title
   text color, border-color, and background tint. No other element in this phase uses
   amber as a decoration.

Semantic risk colors (not accent):
- `--risk-low` (#22c55e / rgba(34,197,94,*)): SAFE verdict, LOW severity flags
- `--risk-medium` (#f59e0b / eab308 in RISK_COLOR): REVIEW verdict, MEDIUM severity flags
- `--risk-high` (#f97316): HIGH severity flags
- `--risk-critical` (#ef4444): BLOCK verdict, CRITICAL severity flags

The amber warning box reuses the medium-risk color values from the established RISK_BG
and RISK_BORDER maps — keeping the same rgba tints that FlagCard already uses for
medium-severity flags.

Source: `globals.css` lines 16–30, `TradeClient.tsx` lines 14–31. CONTEXT.md D-08
(amber warning box intent). RESEARCH.md "Existing Pattern: CSS Design Tokens for
Amber Warning".

### 1. Seller Domain Form Field

Location: inside `TradeForm` grid, after the last existing field ("Commodity").
Rendered via the existing `field()` helper (lines 207–241).

| Property | Value |
|----------|-------|
| Label | "Seller domain (optional)" |

| Hint text | "Used for domain fraud detection — leave blank to skip" |
| Input type | text |
| Required | false (no asterisk, no validation error) |
| FormValues key | `sellerDomain` |
| Submitted as | `sellerDomain: values.sellerDomain.trim() || undefined` |
| Grid position | Full-width row below "Commodity" field, or second column in the 2-col grid |

Interaction states:
- Default: border `1px solid rgba(255,255,255,0.08)`, bg `var(--bg-elevated)`, text 14px/400
- Focus: `outline: 2px solid var(--accent-primary)` (from global `:focus-visible`)
- Filled: identical to default (no special filled state)
- Validation: none — field is optional, blank is valid

Source: CONTEXT.md D-02, RESEARCH.md "GAP-1: Updating TradeForm for sellerDomain field"
(lines 396–430). `field()` helper in TradeClient.tsx lines 207–241.

### 2. Sanction Degradation Warning Box

Location: `ResultsView`, rendered immediately after `<ResultBanner result={result} />`
as a sibling element, before the Actions row and FlagList. Conditionally rendered only
when `result.sanctionDegraded === true`.

| Property | Value |
|----------|-------|
| Background | `rgba(234,179,8,0.08)` (RISK_BG['medium']) |
| Border | `1px solid rgba(234,179,8,0.25)` (RISK_BORDER['medium']) |
| Border radius | 8px (matches FlagCard) |
| Padding | `var(--space-4)` (16px, matches FlagCard) |
| Margin bottom | `var(--space-4)` (16px, before next section) |
| Title text | "Sanction data may be incomplete" |
| Title style | 13px, weight 600, color `#eab308`, margin-bottom 4px |
| Body text | "The OpenSanctions API was unavailable during this check. Cached data was used where available. Manual verification against OFAC, EU FSF, and UN consolidated lists is recommended." |
| Body style | 12px, weight 400, color `var(--text-muted)` (#8a8f98), line-height 1.5 |
| Icon / prefix | None — text-only, no icon library in use |
| Rendered | Only when `result.sanctionDegraded === true` |
| Verdict unchanged | Box is informational only. Verdict (SAFE/REVIEW/BLOCK) and all flags render unchanged beneath it. |

Interaction states:
- No hover, click, or dismiss interaction — it is a static informational banner.
- Respects `prefers-reduced-motion` via global animation rule (no animations used here).

Source: CONTEXT.md D-08. RESEARCH.md "GAP-2: Amber warning box in ResultsView"
(lines 362–392). Design token values from TradeClient.tsx RISK_BG/RISK_BORDER maps
(lines 20–31).

## Copywriting Contract

All copy pre-populated from CONTEXT.md D-08 and RESEARCH.md. No user questions needed.

| Element | Copy |
|---------|------|
| Primary CTA (submit) | "Run Trade Check →" (existing — no change) |
| Form field label | "Seller domain (optional)" |
| Form field hint | "Used for domain fraud detection — leave blank to skip" |
| Warning box title | "Sanction data may be incomplete" |
| Warning box body | "The OpenSanctions API was unavailable during this check. Cached data was used where available. Manual verification against OFAC, EU FSF, and UN consolidated lists is recommended." |
| Empty state (no domain flags fired) | No UI change — domain check silently skips when no domain resolved (D-04) |
| Error state (RDAP failure) | No UI change — RDAP failure silently skips domain check (D-05) |
| Destructive actions | None in this phase |

Copywriting principles applied:
- Tone for warning box: informational, not alarming. The verdict is unaffected; the
  message communicates reduced confidence in the sanction data, not a verdict change.
- "Manual verification recommended" is the explicit call-to-action within the warning.
- No ellipsis, exclamation marks, or alarm language.
- Hint text uses em dash not hyphen for "leave blank to skip" phrasing.

Source: CONTEXT.md D-08 intent ("informational, not alarming"). RESEARCH.md code
example lines 374–384. CONTEXT.md D-04, D-05 (silent skip = no copy needed).

## Interaction States — Complete Map

### Trade Form (pre-submission)

| State | Seller domain field behavior |
|-------|------------------------------|
| Default | Empty, placeholder visible |
| Focus | Accent primary outline ring (2px #5e6ad2) |
| Filled | Text visible, no special treatment |
| Submit — blank | Silently excluded from request body (`undefined`) |
| Submit — filled | Trimmed, sent as `sellerDomain` in JSON body |

### Trade Results (post-submission)

| State | Amber warning box |
|-------|-------------------|
| `sanctionDegraded` absent or false | Box not rendered |
| `sanctionDegraded === true` | Box rendered below ResultBanner, above Actions |
| Domain flags fired (DOMAIN_WHOIS_RISK, DOMAIN_SPOOFING_RISK) | Appear as standard FlagCards in the existing flag list — no special treatment beyond existing FlagCard pattern |
| Domain check skipped (no domain resolved) | No UI change, no notification |

### Loading state

No change to existing LoadingView / GlowLoader. The domain check runs server-side
within `runTradeCheck()` — it does not add a visible loading step.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Form field label association | `field()` helper renders `<label>` wrapping `<input>` — labels are associated by DOM nesting (existing pattern) |
| Optional field indication | "(optional)" in label text — no asterisk, no aria-required |
| Warning box semantics | Plain `<div>` with paragraph text — informational, not a live region. Compliance officers read results after load, so `role="alert"` is not needed (no dynamic injection mid-session). |
| Color alone not used | Warning box title text is `#eab308` (amber) plus "Sanction data may be incomplete" text — color supports text, not replaces it |
| Focus ring | Global `:focus-visible` rule covers the new text input |
| Reduced motion | Global `prefers-reduced-motion` rule covers all animations (warning box has none) |
