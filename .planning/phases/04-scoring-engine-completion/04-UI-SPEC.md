|-------|-------|-------|
| --space-1 | 4px | Icon gaps, bullet indentation, tight inline padding |
| --space-2 | 8px | Compact element spacing, small gaps |

Exceptions: ScoreGauge gauge arc SVG uses `cx = GAUGE_RADIUS + 8` (8px padding hardcoded in component logic, not from token). See Pre-existing Constants section below for additional hardcoded values.

## Typography

Phase 4 typography contract uses exactly 2 font weights: **400** (labels, upgrade prompt text, dimension labels) and **700** (score numeral — pre-existing in ScoreGauge). Four size roles are active.

| Role | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|--------|-------------|---------------|-------|
| Body | 14px | 400 | 22px (1.57) | default | Default prose, descriptions, dimension labels |
| Label | 12px | 400 | 16px (1.33) | -0.13px | Badges, metadata, upgrade prompt text, peripheral score sub-labels, dimension score values |
| Score numeral | 32px | 700 | 1 (unitless) | default | Numeric score inside gauge arc (pre-existing) |
| Mono data | 14px | 400 | 20px | default | Score fractions like `22/25` via `.mono` class (JetBrains Mono) |

Phase 4 typography additions within ScoreGauge:

| Role | Size | Weight | Mapped To | Usage |
|------|------|--------|-----------|-------|
| Score sub-label | 12px | 400 | Label (12px) | "/ 100" beneath numeral; dimension score values (existing) |
| Dimension label | 12px | 400 | Label (12px) | Per-dimension name in breakdown list (existing) |
| Upgrade prompt | 12px | 400 | Label (12px) | Free-user upgrade CTA at bottom of ScoreGauge (new) |

Note: Score sub-label, Dimension label, and Upgrade prompt were previously specified at 11px. They are collapsed into the 12px Label role. These are peripheral, low-emphasis text elements — 12px at weight 400 is functionally equivalent and keeps the project within its declared 4-size typography system.

## Color

All values from `src/styles/globals.css` `:root`. Confirmed design direction: Linear-style dark mode, indigo-violet accent.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#08090a` (--bg-primary) | Page background, outer chrome |
| Secondary (30%) | `#0f1011` (--bg-surface) + `#191a1b` (--bg-elevated) | Sidebar, panels, cards, dropdown surfaces; ScoreGauge track circle, dimension progress track |
| Accent (10%) | `#5e6ad2` (--accent-primary) | Exact elements listed below |
| Destructive | `#ef4444` (--status-listed / --risk-critical) | Sanctioned badge, critical risk arc, destructive actions only |

Accent `#5e6ad2` is reserved for:
1. Focus rings (2px solid --accent-primary) on all interactive elements
2. Upgrade CTA link text inside ScoreGauge pointing to `/pricing`
3. Score arc stroke color when risk level is "Low" (via `getRiskColor()` — uses `--risk-low` #27a644 for low, accent not applied to arc)

Note: The arc stroke color is drawn from the risk-level palette (`--risk-low` green / `--risk-medium` amber / `--risk-high` orange / `--risk-critical` red), NOT from accent. Accent is for brand actions only.

Upgrade prompt link color: `#7170ff` (--accent-violet) — matches existing link `a` color rule.

Shell signal deduction evidence text color: `var(--text-muted)` (#8a8f98) — same pattern as existing evidence strings in ScoreGauge.

## Interaction Contracts

### ScoreGauge — Paywall Boundary

- The dimension breakdown section (bars, labels, fractions, evidence) is conditionally rendered based on `showBreakdown` prop — not blurred/hidden via CSS. No DOM leakage of score values to free users.
- The upgrade prompt renders in the same vertical slot where the dimension list would appear — no layout shift between free/paid views beyond content height difference.
- Upgrade prompt is plain text + a standard `<a>` link styled with `color: var(--accent-violet)`. No button, no modal.

### ScoreGauge — Animation

- Arc transition: `stroke-dashoffset 1.2s cubic-bezier(0.33, 1, 0.68, 1)` triggered on `animated` state (existing).
- Score count-up: cubic ease-out over 1200ms via requestAnimationFrame (existing).
- Dimension bar expand: `width Xs cubic-bezier(0.33, 1, 0.68, 1) {0.2 + index * 0.08}s` (staggered by bar index) — only renders when `showBreakdown === true`.
- All animations disabled when `prefers-reduced-motion: reduce` — existing `@media` rule in globals.css covers this.

### Dimension Evidence Bullets

- Evidence strings are rendered as `<ul>` with `<li>` items inside each triggered dimension row.
- Bullet marker is `·` (middle dot) rendered as `aria-hidden="true"` `<span>` positioned absolutely at left: 0 — existing pattern preserved.
- Shell signal evidence strings are appended to `entityExistence.evidence[]` array — same data shape, no new rendering logic needed.

### Focus Management

- No modal dialogs or focus traps introduced in Phase 4.
- Upgrade CTA link uses standard anchor focus: `outline: 2px solid var(--accent-primary); outline-offset: 2px` via global `:focus-visible` rule.

|-------------|---------------|
