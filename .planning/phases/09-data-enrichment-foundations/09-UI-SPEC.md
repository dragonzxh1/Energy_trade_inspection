## Spacing Scale

All values from existing globals.css `--space-*` tokens. No new tokens required.

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, dot indicator margin |
| --space-2 | 8px | Badge internal padding, gap between badges |
| --space-3 | 12px | Alert item row padding (vertical), icon-text gap |

## Component Specifications

### 1. FraudAlertsPanel

A Server Component panel rendered inside the TabNav tab system, wrapped in `<ContentLock>` (F3 gate — Starter+ only), consistent with DirectorsPanel and IntelligencePanel placement.

#### Placement

**Company detail page (`/company/[slug]`):**
- New tab inserted between "Risk Flags" and "Offshore Leaks"
- Tab object: `{ id: 'fraud-alerts', label: 'Fraud Alerts' }`
- Panel wrapped in `<ContentLock unlocked={f3Unlocked} reason={lockReason}>`

**Vessel detail page (`/vessel/[imo]`):**
- New tab inserted between "Risk Flags" and "PSC History"
- Tab object: `{ id: 'fraud-alerts', label: 'Fraud Alerts' }`
- Panel wrapped in `<ContentLock unlocked={f3Unlocked} reason={lockReason}>`

#### Card shell (matches existing `card` style exactly)

```
backgroundColor: var(--bg-surface)
borderRadius: 10px
padding: var(--space-5)
border: 1px solid var(--border-subtle)
```

#### Panel header

```
Section title: "Fraud Alerts" (blacklist count > 0) or "Fraud Alerts (N)" showing total count
Style: sectionTitle pattern — 11px, weight 600, uppercase, letter-spacing 0.08em, color var(--text-muted), margin-bottom var(--space-4)
```

#### States

**Loading state:**
- Render the card shell with section title "Fraud Alerts"
- Body: 3 skeleton rows — each is a 40px-tall `<div>` with `backgroundColor: var(--bg-elevated)`, `borderRadius: 6px`, `opacity: 0.5`, `marginBottom: var(--space-3)`
- No animation (no existing skeleton animation in codebase; keep it static)

**Empty state (no matches):**
```
Copy: "No fraud alerts on record for this entity."
Style: color var(--text-muted), fontSize 14px, textAlign center, padding var(--space-8) 0
Sub-copy: "Covers Rotterdam Port Blacklist, FuelScamAlert, Ametheus, and other industry sources."
Sub-copy style: fontSize 12px, color var(--text-muted), textAlign center, marginTop var(--space-2)
```

**Populated state:**
- List of alert items, each separated by `borderBottom: 1px solid var(--border-subtle)`
- Last item: no bottom border
- Items sorted: blacklist entries first, then whitelist; within each group sorted by `synced_at DESC`

#### Alert item layout

```
display: flex
justifyContent: space-between
alignItems: flex-start
padding: var(--space-3) 0
borderBottom: 1px solid var(--border-subtle)
```

Left column (flex: 1):
- Row 1: Entity name + source badge (flex row, gap var(--space-2), flex-wrap wrap)
  - Entity name: `company_name` from DB — 14px, weight 500, color var(--text-primary)
  - Source badge: see Source Badge spec below
- Row 2: Metadata line — `fraud_type` (if available) + `synced_at` date
  - 12px, color var(--text-muted), marginTop 2px
  - Format: `{fraud_type label} · Reported {date}` — or just `Reported {date}` if no fraud_type
  - Date format: `Jan 15, 2025` (en-US, year + short month + day)
- Row 3 (conditional): description — only if `description` is non-null and non-empty
  - 12px, color var(--text-muted), fontStyle italic, marginTop 4px
  - Truncate to 120 characters with `…` if longer
- Row 4 (conditional): scam URL — only if `scam_url` is non-null
  - Prefix label "Fake site:" in var(--text-muted), then URL in var(--accent-primary), 11px
  - `target="_blank" rel="noopener noreferrer"`

Right column (flexShrink: 0, marginLeft: var(--space-3)):
- List type badge: "BLACKLIST" or "WHITELIST"
  - fontSize 11px, fontWeight 600, textTransform uppercase, letterSpacing 0.06em
  - Blacklist: color #f59e0b, backgroundColor rgba(245,158,11,0.1)
  - Whitelist: color #10b981, backgroundColor rgba(16,185,129,0.1)
  - padding: 2px 6px, borderRadius 4px

#### Source badge (inline, next to entity name)

Pattern mirrors the dataset badge in OffshoreLeaksPanel.

```
fontSize: 10px
fontWeight: 600
textTransform: uppercase
letterSpacing: 0.06em
color: var(--accent-amber)       /* #f59e0b */
backgroundColor: rgba(245,158,11,0.1)
border: 1px solid rgba(245,158,11,0.15)
padding: 1px 6px
borderRadius: 4px
```

Source display name map (machine key → display label):

| source key | Display label |
|------------|---------------|
| storagespoofing | Rotterdam Port |
| fuelscamalert | FuelScamAlert |
| ametheus | Ametheus |
| glo-innovations | Glo-Innovations |
| capitalgaslogistics | Capital Gas |
| (unknown) | source key uppercased |

#### Panel footnote

Render after the item list:

```
Copy: "Fraud alerts sourced from industry blacklists. Independent of government sanctions lists. Verify against primary sources before taking action."
Style: fontSize 11px, color var(--text-muted), lineHeight 16px, marginTop var(--space-4)
```

If `source_url` is available for any alert, add a "View Source" link after the footnote:
```
Link style: color var(--accent-primary), fontSize 11px, textDecoration none
```

### 2. ICIJ Sanctioned Node Indicator (Phase 9 groundwork for Phase 10)

This is a **visual groundwork element** — a small indicator badge on existing ICIJ match rows in `OffshoreLeaksPanel` to show when a matched ICIJ entity has `is_sanctioned = true`.

Phase 9 does NOT render the full network graph (that is Phase 10). Phase 9 only marks the data and adds a small inline indicator on existing rows.

#### Indicator placement

On each row in `OffshoreLeaksPanel` (company page) — right side, between match confidence and ICIJ link:

```
Condition: icij_entity.is_sanctioned === true (from joined data)
Element: inline badge reading "Sanctioned Entity"
```

#### Indicator style

```
fontSize: 10px
fontWeight: 600
textTransform: uppercase
letterSpacing: 0.06em
color: var(--status-listed)          /* #ef4444 */
backgroundColor: rgba(239,68,68,0.12)
border: 1px solid rgba(239,68,68,0.2)
padding: 2px 6px
borderRadius: 4px
marginLeft: var(--space-2)
```

#### When `is_sanctioned = false` or column not yet populated

Do not render the indicator. Absence of badge = not sanctioned (no false-positive noise).

#### `sanctions_match` tooltip

If `sanctions_match` is non-null, show it via native `title=` attribute on the badge:
```
title={`Matched: ${sanctions_match}`}
```

This follows the existing WarningBadge tooltip convention (`title=` only, no custom tooltip component).

## Responsive Behavior

ETI is a data-dense desktop tool. Mobile layout follows existing patterns:

| Breakpoint | Behavior |
|------------|----------|
| > 900px | Full entity-layout two-column (aside + main), existing `.entity-layout` class |
| ≤ 900px | Single column stack (existing CSS handles this) |
| ≤ 600px | Alert item layout: stack left and right columns vertically (flexDirection column, right column aligns left, marginTop var(--space-2)) |
| Touch targets | Alert rows: min-height 44px on mobile |

No new breakpoints. The existing `.entity-layout` responsive CSS handles page layout.
Alert items collapse to vertical stack at ≤ 600px — identical to how existing data-row items collapse.

## Implementation Notes for Executor

1. `FraudAlertsPanel` is a **Server Component** — data fetched server-side in `getCompanyFraudAlerts(entityName)` and `getVesselFraudAlerts(operatorName, managerName)` repository functions (to be created in Phase 9 plan).

2. The panel receives pre-fetched `FraudAlert[]` prop — no client-side fetch, no loading spinner needed for the panel itself. The loading state described above is for the tab reveal (tab switching is client-side via TabNav which already handles display:none toggling).

3. Alert item rows use `className="data-row"` (existing CSS hover class) for consistency.

4. The ICIJ sanctioned indicator requires the repository to JOIN `icij_entities.is_sanctioned` and `icij_entities.sanctions_match` when building `IcijMatch` objects. The `IcijMatch` TypeScript type needs two new optional fields: `isSanctioned?: boolean` and `sanctionsMatch?: string | null`.

5. Tab insertion order — company page tabs array becomes:
   ```
   Registration | Directors | Beneficial Owners | Vessels | Risk Flags | Fraud Alerts | Offshore Leaks | Intelligence | Domain | Sources
   ```
   Vessel page tabs array becomes:
   ```
   Vessel Details | AIS Tracking | Draft Risk | Risk Flags | Fraud Alerts | PSC History | Intelligence | Sources
   ```
