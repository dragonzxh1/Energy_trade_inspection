# Sketch 001: Controls Quality

## Design Question
How should buttons and inputs feel? This sketch explores three distinct "texture philosophies" for the control vocabulary — before applying them across the real UI.

## How to View
```
open .planning/sketches/001-controls-quality/index.html
```

## Variants
- **A: Sharp & Flat** — Clean border-defined buttons with flat fills, hover lift (+1px translateY + glow). Focus states use thin ring. Precise and minimal.
- **B: Textured & Dimensional** — Micro-gradient buttons (top lighter, bottom darker), inset-shadow inputs that feel "pressed in", gradient score number. Feels expensive.
- **C: Muted & Clinical** — Low-saturation primary buttons, focus-outline-driven inputs (2px outline, no glow), ultra-minimal score card. Bloomberg terminal energy.

## What to Look For
- How much visual weight should the primary button carry?
- Do inputs feel like they belong in a form, or just float on the page?
- Which treatment makes the score card number feel authoritative vs decorative?
- Does the gradient score text (B) feel right for a compliance tool?
