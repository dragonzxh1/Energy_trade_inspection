Accent (`--accent-primary`) reserved for: focus rings on interactive elements, brand scan-line overlay. Not used on warning badges.

### Warning Badge Color Contract (Phase 2 new tokens)

Amber/orange family — distinct from red (sanctioned) and green (clear). Uses the pre-declared `--accent-amber` token and derives from the existing `--risk-medium` amber.

| Element | Text color | Background | CSS Variable |
|---------|------------|------------|-------------|
| WarningBadge text | `#f59e0b` | `rgba(245, 158, 11, 0.12)` | text: `--accent-amber`; bg: derived |
| WarningBadge border | none | — | No border — consistent with SanctionBadge pattern |

Rationale: `--accent-amber: #f59e0b` is already declared in `globals.css` with the comment "Amber — warnings, Phase 2 pending." This is the pre-designated warning color. Background opacity `0.12` matches the established pattern (`rgba(16, 185, 129, 0.12)` for clear, `rgba(239, 68, 68, 0.12)` for listed).

No new CSS variables are introduced. The `WarningBadge` component uses inline styles referencing `var(--accent-amber)` and the literal rgba value — same pattern as `SanctionBadge`.

## Component Contract: WarningBadge

### Purpose

Displays a per-regulator amber warning badge on entity pages. One badge instance per warning list hit. Multiple badges stack horizontally when an entity appears on multiple warning lists.

### Props Interface

```typescript
interface WarningBadgeProps {
  source: string         // machine key: 'fca' | 'finma' | 'sfc' | 'mas' | 'dfsa' | 'sca' | 'cma'
  sourceName: string     // human-readable: e.g. 'FCA (UK)', 'MAS (Singapore)'
  jurisdiction: string   // ISO region: e.g. 'UK', 'CH', 'HK', 'SG', 'AE-DU', 'AE', 'OM'
  size?: 'sm' | 'md'    // default: 'md'
}
```

### Badge Label Format

Label text: `{SOURCE_ABBR} · {JURISDICTION_SHORT}`

| source | Label displayed |
|--------|-----------------|
| `fca` | `FCA · UK` |
| `finma` | `FINMA · CH` |
| `sfc` | `SFC · HK` |
| `mas` | `MAS · SG` |
| `dfsa` | `DFSA · Dubai` |
| `sca` | `SCA · UAE` |
| `cma` | `CMA · Oman` |

Separator: `·` (middle dot, U+00B7) with a single space on each side. Not a pipe, not a slash.

### Tooltip Contract

- Trigger: `title` attribute on the wrapping `<span>` (matches existing Badge primitive pattern — no JS tooltip library needed in Phase 2)
- Content: `{sourceName} — Regulatory Warning List`
- Example: `FCA (UK) — Regulatory Warning List`
- Tooltip is purely informational — no action affordance

### Stacking Layout

- Container: `display: flex; flex-wrap: wrap; gap: var(--space-2)` (8px)
- Placement on entity pages: directly below `SanctionBadge` in the sidebar, with `marginTop: var(--space-2)` (8px) between the sanction badge group and the warning badge group
- On mobile (`max-width: 560px`): inherits the existing `.entity-sidebar-badges` flex-wrap behavior — badges wrap naturally

### Render condition

- Render the warning badge group only when `warningHits.length > 0`
- When `warningHits.length === 0`: render nothing (no badge, no empty state message — absence of badge = no warning)
- No "Regulatory Warning: None" badge — silence is the negative signal

### No animation

- WarningBadge does NOT get a glow/pulse animation class
- Rationale: Amber warnings are informational, not critical. Red sanction badge pulses (`badge-glow-listed`); green clear badge pulses (`badge-glow-clear`). Amber is static — steady state, not alerting
- Reduced motion: already handled globally by `@media (prefers-reduced-motion: reduce)` — no per-badge handling needed

### Integration points (exact locations)

All three entity pages: insert warning badge group immediately after the `<div style={{ marginTop: 'var(--space-3)' }}>` block containing `<RiskBadge />`.

`src/app/company/[slug]/page.tsx` — line ~823 area:
```tsx
{/* Warning Badges — after RiskBadge */}
{warningHits.length > 0 && (
  <div style={{ marginTop: 'var(--space-2)', display: 'flex', flexWrap: 'wrap', gap: 'var(--space-2)' }}>
    {warningHits.map(hit => (
      <WarningBadge key={hit.source} source={hit.source} sourceName={hit.source_name} jurisdiction={hit.jurisdiction} />
    ))}
  </div>
)}
```

Same pattern for `src/app/vessel/[imo]/page.tsx` and `src/app/terminal/[id]/page.tsx`.

## Interaction States

This phase introduces no new interactive elements beyond the badge tooltip. All states:

| State | Behavior |
|-------|----------|
| Default | Amber badge rendered, static (no animation) |
| Hover | `title` tooltip appears (native browser) — no hover style change on badge |
| Focus | `outline: 2px solid var(--accent-primary); outline-offset: 2px` — inherited from global `:focus-visible` |
| Reduced motion | No effect (badge has no animation) |
| Multiple hits | All badges render side-by-side, flex-wrap on overflow |
| Zero hits | Badge group container not rendered |
| Data loading | Entity page uses existing server component pattern — no skeleton state needed for warning badges specifically; page loads with all data |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Color alone does not convey meaning | Badge label text (`FCA · UK`) provides textual identification — amber color is supplementary |
| Screen reader identification | `title` attribute on badge span provides tooltip text to screen readers |
| Keyboard focus | Badges are `<span>` elements — not natively focusable. If aria compliance is needed, wrap in `<button>` with `tabIndex={0}` and `aria-label`. Phase 2 uses span + title for simplicity |
| Contrast ratio | Amber `#f59e0b` on `rgba(245, 158, 11, 0.12)` bg over `#08090a` base: text contrast against dark background passes WCAG AA (4.5:1 minimum) — `#f59e0b` on near-black exceeds 7:1 |
| Reduced motion | Global `@media (prefers-reduced-motion: reduce)` in `globals.css` — no badge-specific handling needed |
